# 4_党群工作部_党务工作智能问答助手_MVP需求拆解

## 1. 页面信息架构

1. 工作台首页（Dashboard）
- 核心统计：知识条目数、高优先级待办、闭环进度。
- 运营洞察：重复咨询压降率、命中率、推荐采纳率。
- 服务留痕：近期问答处理事件时间线。

2. 知识库构建（Workspace）
- 条目列表：编号、名称、类型、领域、范围、状态。
- 新增表单：条目类型、责任岗位、结构化摘要。
- 状态推进：待校验 -> 问答服务中 -> 推荐优化中 -> 已发布。

3. 问答服务运营（Workspace）
- 任务列表：咨询主题、优先级、受理岗位、处理状态。
- 新增问答任务：标准答复 + 推荐文件。
- 删除与推进：二次确认 + 操作反馈。

4. 个性化推荐（Analysis）
- 覆盖分析：不同知识类型条目分布。
- 推荐策略：岗位化学习建议和运营提示。
- 近期留痕：展示最近处理动作。

5. 党务智能问答助手（Assistant）
- 会话历史、对象选择、自然语言问答。
- 结果采纳写回：采纳后同步状态和留痕。

6. 系统治理模块
- 用户管理、日志审计、系统设置。

## 2. 核心业务对象

1. 党务知识条目
- 字段：编号、名称、类型、业务领域、适用范围、优先级、责任岗位、状态。

2. 问答会话
- 字段：会话ID、提问、结果、来源模式、采纳状态、保存摘要。

3. 操作留痕
- 字段：模块、动作、对象、操作人、结果、摘要、时间。

4. 系统配置
- 字段：分组、配置键、配置值、启停状态、更新人、更新时间。

## 3. MVP功能边界

1. 已包含
- 结构化知识条目管理
- 问答任务处理与状态流转
- AI问答与结果采纳
- 岗位化推荐分析展示
- 用户/审计/设置治理页面

2. 暂不包含
- 真实政策抓取接口
- 统一身份认证对接
- 实时消息推送
- 多租户隔离与复杂权限模型

## 4. 关键交互流程

1. 知识条目创建流程
- 点击“新增知识条目” -> 填写表单 -> 创建成功 -> 条目进入“待校验” -> 出现在知识库列表。

2. 问答任务推进流程
- 点击“推进处理” -> 高优先级任务触发二次确认 -> 状态向下一阶段推进 -> 写入服务留痕。

3. 删除流程
- 点击“删除” -> 弹窗确认 -> 删除成功 -> 列表数量减少 -> 留痕同步更新。

4. 智能问答采纳流程
- 选择知识条目 -> 输入问题 -> 生成建议 -> 点击采纳 -> 状态更新并写入保存摘要。

## 5. 数据字段映射（MVP）

1. 知识库构建表单
- `title`：知识条目名称
- `modelType`：条目类型（政策解读/流程指引/案例答疑）
- `industry`：业务领域
- `region`：适用范围
- `owner`：责任岗位
- `inputParams`：结构化摘要

2. 问答运营表单
- `title`：咨询主题
- `owner`：受理岗位
- `riskLevel`：服务优先级（高/中/低）
- `outputResult`：标准答复
- `prediction`：推荐文件/学习建议

3. 智能问答结果
- `resultType`：党务问答建议
- `saveStatus`：待采纳/已采纳
- `saveSummary`：采纳后摘要留痕

## 6. 演示验收口径

- 能完成一次完整“创建 -> 推进 -> 问答采纳 -> 留痕回看”闭环。
- 所有列表、卡片、会话内容均为专业业务 Mock 数据，无 test1/test2 占位。
- 关键操作均有 Loading、成功/失败反馈、必要二次确认。
